ТЕМА: ОПРЕДЕЛЕНИЕ ФУНКЦИИ
	Преимущества функции:
	-Упрощение
	-Повторное использование
	-Имя Функции говорит о её цели
	-Единственный способ сделать рекурсию
	У функции есть:
	-Прототип (Это как её заголовок, мы её обозначаем и записываем от первой строчки кода функции до последней) (int cybe (double x){})
	-Определение функции (что она делает)(int cube (double x) { return x*x*x;})
	-Вызов функции (непосредственный запуск функции) (result = cube(x))
	[класс] тип имя (параметры) {  ....  }
	Классы:
	-inline (для всех файлов)
	-static (не для всех файлов которые существуют)
	Функция работает только ДЛЯ ОДНОЙ ЗАДАЧИ, не надо делать в функции что-то параллельно.
	Объявление функций: 
		-В одном файле над main() полностью(прототип, определение), либо после main() определение функции а над main() только прототип
		-В другом файле принадлежащему тому же самому проекту через #include "anothfile.cpp"
	В многофайловых проектах компилируется только изменённый cpp файл 
	Если h и cpp вызимосвязаны, то изменения в h ведёт к пересборке проекта.
	Правильная реализация состоит в следующем: в хедере прототип делаем только, а в cpp реализация (и в include выбираем название хедера .h)
	Возможности функции:
		Возврат нескольких переменных - variant 
	Методы передачи параметров в функции:
		-Стек - когда в программе запускается очередная функция то на стеке резервируется блок памяти, достаточный для хранения её переменных. Вершина стека растёт в сторону уменьшения адресов. Этот механизм возмонжен благодаря тому, что в каждой функции предусмотрен механизм возврата.
		void sum (int a , int b){int c = a + b;}
		int main () {int x = 0;
		int y = 0;
		sum(x,y)} после выхода из функции sum  мы из памяти для a и b,c перемещаемся в память main где у нас x и y
		-ЭТО СПОСОБ ПЕРЕДАЧИ ПАРАМЕТРА ФУНКЦИИ ПО ЗНАЧЕНИЮ
		-А ПО ССЫЛКЕ БУДЕТ ПАМЯТЬ HEAP (куча) память здесь не копируется как в стеке и ничего не перезаписывает (int& a (адрес переменной))
